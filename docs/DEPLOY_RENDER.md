# Deploy Backend On Render

This file is the fastest path to deploy Asgard backend on Render free tier.

## 1) Create service from blueprint

1. In Render, choose **New +** -> **Blueprint**.
2. Select this repository: `ayuush012/personal-agent-suite`.
3. Render detects `render.yaml`.
4. Confirm service creation.

Render will create:

- Service name: `asgard-backend`
- Runtime: Python
- Build command: `pip install -r requirements.txt`
- Start command: `uvicorn backend.main:app --host 0.0.0.0 --port $PORT`
- Health path: `/health`

## 2) Set required secret

After the service is created, add one manual environment variable in Render:

```bash
GROQ_API_KEY=your_real_groq_key
```

`SECRET_KEY` is auto-generated by `render.yaml`.

## 3) Verify deployment

When deployment finishes, check:

```bash
https://<your-render-service>.onrender.com/health
```

Expected response:

```json
{"status":"ok","app":"asgard"}
```

## 4) Connect frontend to backend

In your frontend host env variables:

```bash
VITE_API_URL=https://<your-render-service>.onrender.com
VITE_WS_URL=wss://<your-render-service>.onrender.com
```

Redeploy frontend after setting these.

## Notes

- Render free tier sleeps on inactivity, so first request after idle may take longer.
- Current backend state is in-memory (no persistent DB required for v1 recruiter demo).
